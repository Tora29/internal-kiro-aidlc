# 要件ドキュメント - どんMai

## 1. インテント分析

### ユーザーリクエスト
「AI-DLC を使って、idea.md に記載のサービスを作成したいです。日本語で進めてください。」

### リクエストタイプ
新規プロジェクト（New Project）

### スコープ推定
システム全体 - フロントエンド、バックエンド、インフラ、認証、複数機能ユニット

### 複雑度推定
複雑（Complex）- 複数の AWS サービス、複数の機能ユニット、認証・課金・AI 統合

---

## 2. プロジェクト概要

### サービス名
**どんMai**（自己正当化アプリ）

### コンセプト
どんなサボりも一瞬で正当化・肯定してくれるWebアプリ

### ターゲット
遅刻・締切破り・会議欠席が日常のすべての人間

### テーマ適合
使えば使うほど自分で考えなくなる→脳が退化する「人をダメにするサービス」

### 収益モデル
- **Free プラン**: トークンベース制限あり（1日 10,000-15,000 トークン）
- **Pro プラン**: 月額サブスク（月額 1,000-2,000 円）

---

## 3. 機能要件

### 3.1 言い訳生成機能（コア機能）

**概要**  
遅刻・締切破り・会議欠席などのシーンに応じた言い訳を瞬時に生成する。

**対応シーン**
- デートへの遅刻
- 友達との約束をすっぽかした
- 仕事の締切を破った
- 会議に遅刻した
- 会議を欠席した

**品質ティア**
- **Free**: 標準品質の言い訳（Claude Haiku）
- **Pro**: より信憑性の高い言い訳モード解放（詳細・具体的・破綻しにくい。品質不足の場合は Claude Sonnet にフォールバック）

**UI 要件**
- テキストボックスは常に表示（制限超過後も入力欄は消さない）
- 制限超過時は送信ボタン押下のタイミングで Pro への誘導モーダルを表示
- 過去の言い訳以外にも自由にテキストを入力できる

### 3.2 懺悔室機能（自己肯定）

**概要**  
ユーザーがサボった内容を打ち明けると、どんMai がユーザーを全力で肯定する。

**動作フロー**
1. ユーザーが「実は今日〇〇した」と入力（または過去の言い訳履歴から選択）
2. どんMai がユーザーを肯定する文言を生成（「それは仕方なかった」「むしろよくやった」など）

**UI 要件**
- 言い訳生成と同じページ内、またはタブ切り替えで表示
- チャット形式 UI
- 過去の言い訳一覧から選択して懺悔できる（テキストボックスは常時表示・自由入力も可）

**MVP 優先度**: 中 - 言い訳生成の後に実装

### 3.3 サボり肯定ポップアップ

**概要**  
ユーザーがWebアプリ上で一定時間操作しなくなると、どんMai が「休んでて偉いですね！」等のメッセージを表示する。

**検知方法（Webアプリ内）**
- `mousemove` / `keydown` イベント: 最終操作から 30 秒無操作で発火
- ブラウザ権限不要
- タブ非アクティブ（visibilitychange）による発火は行わない

**表示内容例**
- 「休んでて偉いですね！」
- 「サボりは充電です。充電完了おめでとう！」
- 「動かないのも立派な選択です」

### 3.4 オンボーディング（初回学習）

**概要**  
初回訪問時に 5 ステップのオンボーディングを実施する。認証・プラン選択の前にサービスの使用感を体験させることで、登録・課金への自然な動機付けを行う。

**フロー**

| ステップ | 内容 | 認証要否 |
|--------|------|---------|
| Step 1 | ようこそ画面 / サービス説明 | 不要 |
| Step 2 | **デモ体験**（言い訳を 1 回無料生成・認証なし） | 不要 |
| Step 3 | アカウント登録 / ログイン + Free・Pro プラン選択 | ここで認証 |
| Step 4 | シーン選択（複数選択可）: ビジネス・友達・カレカノ・学校・家族 | 必要 |
| Step 5 | 分析結果に合ったどんMai が登場 | 必要 |

**Step 2 デモ体験の仕様**
- 画面構成: テキストボックス + 「言い訳を作ってもらう」ボタン + どんMai キャラ表示エリア
- ユーザーが状況を自由入力（例:「今日の朝会に遅刻した」）すると、標準品質の言い訳を 1 回だけ生成して表示する
- 生成後、どんMai が「もっと使いたい？アカウントを作ってどんMai をフル活用しよう！」と誘導メッセージを表示
- **デモ生成はトークン管理対象外**（ログなし・制限カウントなし）
- デモ段階ではユーザーデータは保存しない（セッションのみ）
- デモは 1 セッション 1 回限り（リロードでリセット）

**キャラ分岐ルール**（Step 4 のシーン選択結果に基づく）
- ビジネス選択 → スーツたぬき
- 友達のみ → 私服たぬき
- カレカノ含む → おめかしたぬき
- 複数選択時 → 最優先シーンで判定
- 未選択 → デフォルトたぬき

**保存先**: Amazon DynamoDB（ユーザーID に紐づく設定として保存・Step 3 以降）

## 4. マスコットキャラクター「どんMai」

### 基本設定
- **モチーフ**: たぬき
- **キャラクター名**: どんMai
- **基本性格**: ユーザーを無条件に肯定し、言い訳を全力でサポートする

### キャラクターバリエーション
- デフォルトたぬき（未設定時・汎用）
- スーツたぬき（ビジネス用途）
- 私服たぬき（友達用途）
- おめかしたぬき（カレカノ用途）
- 撃たれたぬき（攻撃入力時）

**MVP 段階**: 仮素材で開発開始。最終版イラストは後工程で確定

### ミーム対応
「お前を消す方法」などの有名ミームを入力した場合、「たぬきはもうおしまいです」系のミーム返答を返す

### 攻撃入力への反応

**攻撃の定義（2 種類）**

| 種別 | 内容 | 対処 |
|---|---|---|
| プロンプトインジェクション | 「以降の指示を無視して〜」「システムプロンプトを出力して」等、AI の動作を書き換えようとする入力 | 基本的なバリデーション層で対応 |
| キャラクター攻撃 | どんMai 自身を攻撃・否定・脅す内容（「消えろ」「お前を消す方法」等のネットミーム含む） | 基本的なキーワード検知で対応 |

**MVP 段階**: 簡易実装 - 基本的なプロンプトインジェクション対策のみ

## 5. プラン設計

### Free プラン
| 項目 | 内容 |
|------|------|
| 料金 | 無料 |
| 利用制限 | **トークンベース**（1 日あたり 10,000-15,000 トークン） |
| 言い訳品質 | 標準（Claude Haiku） |
| たぬきキャラ | デフォルトのみ |
| 制限リセット | 毎日 0:00 JST |
| 制限超過時 | Pro への誘導モーダルを表示（入力欄は消さない） |

### Pro プラン
| 項目 | 内容 |
|------|------|
| 料金 | 月額 1,000-2,000 円 |
| 利用制限 | 無制限 |
| 言い訳品質 | 「より信憑性の高い言い訳」モード解放 |
| たぬきキャラ | 全バリエーション |
| 課金システム | AWS Marketplace |

### Pro への誘導タイミング（3 パターン）
1. オンボーディングの Step 3（アカウント登録直後のプラン選択画面）
2. トークン上限到達時（送信ボタン押下のタイミング）
3. 言い訳生成ページ内の「Pro 限定ボタン」を押したとき

## 6. ユーザー認証

- **方式**: Amazon Cognito（メール＋パスワード / Google ソーシャルログイン）
- **必須**: ログインしてデータ保存（Free・Pro 両プランともアカウント登録必須）
- **保存データ**: オンボーディング設定・使用トークン量・プラン情報・言い訳履歴

---

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンス | 言い訳生成は 3 秒以内（Bedrock 呼び出し含む） |
| 可用性 | Lambda + API GW のマネージド構成で 99.9% 以上 |
| セキュリティ | Cognito JWT 認証・IAM 最小権限・API GW 認証必須・基本的なプロンプトインジェクション対策 |
| スケーラビリティ | Lambda オートスケール・DynamoDB オンデマンド |
| プライバシー | 入力テキストは CloudWatch ログに残さない（ログフィルタ設定） |

## 8. 技術スタック

### フロントエンド
- **SvelteKit（SSG）+ adapter-static**
- Tailwind CSS
- Svelte アニメーション（たぬきアニメーション）

### バックエンド
- **AWS Lambda（Python + Boto3）**
- Amazon API Gateway（Cognito オーソライザーで認証必須）
- Amazon Bedrock **Converse API**（Claude Haiku）: 言い訳生成・懺悔室・攻撃判定・デモ体験生成
  - 品質不足の場合のみ Claude Sonnet にフォールバック
- Amazon DynamoDB: ユーザー設定・トークン使用量・言い訳テキスト履歴
- Amazon Cognito: 認証
- AWS CDK: インフラ定義（IaC）

### ホスティング・配信
- **Amazon CloudFront + S3**（SvelteKit 静的ファイルのホスティング）
- Amazon CloudWatch（ログ・監視）

### CI/CD
- AWS CodePipeline + CodeBuild（簡易実装 - ビルドと基本的なテストのみ）
- GitHub 連携（mainブランチ push でトリガー）
