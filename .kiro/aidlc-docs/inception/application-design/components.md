# コンポーネント定義 - どんMai

## コンポーネント概要

「どんMai」は、レイヤーベースのアーキテクチャで設計されています。フロントエンド層、バックエンド層、データ層に分離されています。

---

## フロントエンド層コンポーネント

### 1. ランディングページコンポーネント
**責務**: サービス説明とデモへの導線を提供

**主要機能**:
- サービス説明の表示
- 「デモを試す」ボタンの表示
- ランディングページのレイアウト管理

**インターフェース**:
- 入力: なし
- 出力: HTML/CSS（ランディングページ）

---

### 2. デモ体験コンポーネント
**責務**: 認証なしで言い訳生成を 1 回体験させる

**主要機能**:
- テキストボックスの表示
- 言い訳生成ボタンの表示
- 言い訳の表示
- 「アカウントを作成」ボタンの表示

**インターフェース**:
- 入力: ユーザーの状況テキスト
- 出力: 生成された言い訳

---

### 3. オンボーディングコンポーネント
**責務**: 5 ステップのオンボーディングフローを管理

**主要機能**:
- Step 1: ようこそ画面
- Step 2: デモ体験
- Step 3: アカウント登録・プラン選択
- Step 4: シーン選択
- Step 5: キャラクター表示

**インターフェース**:
- 入力: ユーザーの選択（プラン、シーン等）
- 出力: オンボーディング完了状態

---

### 4. 言い訳生成UIコンポーネント
**責務**: 言い訳生成機能の UI を提供

**主要機能**:
- テキストボックスの表示
- 「言い訳を作ってもらう」ボタンの表示
- 生成された言い訳の表示
- 過去の言い訳履歴の表示
- Pro 誘導モーダルの表示

**インターフェース**:
- 入力: ユーザーの状況テキスト、トークン使用量
- 出力: 生成された言い訳、Pro 誘導モーダル

---

### 5. 懺悔室UIコンポーネント
**責務**: 懺悔室機能の UI を提供

**主要機能**:
- チャット形式の UI
- テキストボックスの表示
- 肯定メッセージの表示
- 過去の言い訳一覧の表示

**インターフェース**:
- 入力: ユーザーの懺悔テキスト
- 出力: 肯定メッセージ

---

### 6. サボり肯定ポップアップコンポーネント
**責務**: 無操作検知とポップアップ表示を管理

**主要機能**:
- `mousemove` / `keydown` イベント監視
- 30 秒無操作検知
- ポップアップの表示
- ポップアップの閉じる機能

**インターフェース**:
- 入力: ユーザーの操作イベント
- 出力: ポップアップ表示

---

### 7. キャラクター表示コンポーネント
**責務**: どんMai キャラクターの表示と管理

**主要機能**:
- キャラクターの表示（デフォルト、スーツ、私服、おめかし）
- キャラクターアニメーション
- キャラクター選択ロジック

**インターフェース**:
- 入力: シーン選択結果、ユーザープラン
- 出力: 表示するキャラクター

---

### 8. 認証UIコンポーネント
**責務**: ログイン・登録画面の UI を提供

**主要機能**:
- ログインフォームの表示
- 登録フォームの表示
- Google ソーシャルログインボタンの表示
- エラーメッセージの表示

**インターフェース**:
- 入力: ユーザーの認証情報
- 出力: 認証トークン

---

### 9. 設定UIコンポーネント
**責務**: ユーザー設定画面の UI を提供

**主要機能**:
- プラン情報の表示
- トークン使用量の表示
- キャラクターリセットボタン
- ログアウトボタン

**インターフェース**:
- 入力: ユーザー情報、プラン情報
- 出力: ユーザーの設定変更

---

### 10. 共通UIコンポーネント
**責務**: 再利用可能な UI コンポーネントを提供

**主要機能**:
- ボタンコンポーネント
- フォームコンポーネント
- モーダルコンポーネント
- ナビゲーションコンポーネント
- ローディングスピナー

**インターフェース**:
- 入力: コンポーネント固有のプロパティ
- 出力: レンダリングされた UI

---

## バックエンド層コンポーネント

### 11. 認証サービス
**責務**: ユーザー認証と認可を管理

**主要機能**:
- Cognito との連携
- JWT トークンの検証
- ユーザーセッション管理

**インターフェース**:
- 入力: ユーザー認証情報
- 出力: JWT トークン

---

### 12. 言い訳生成サービス
**責務**: 言い訳を生成する

**主要機能**:
- Amazon Bedrock との連携
- Claude Haiku/Sonnet の選択
- プロンプトエンジニアリング
- トークン使用量の計算

**インターフェース**:
- 入力: ユーザーの状況テキスト、プラン情報
- 出力: 生成された言い訳

---

### 13. 懺悔室サービス
**責務**: 肯定メッセージを生成する

**主要機能**:
- Amazon Bedrock との連携
- Claude Haiku による肯定メッセージ生成
- プロンプトエンジニアリング

**インターフェース**:
- 入力: ユーザーの懺悔テキスト
- 出力: 肯定メッセージ

---

### 14. トークン管理サービス
**責務**: ユーザーのトークン使用量を管理

**主要機能**:
- トークン使用量の記録
- 1 日ごとのリセット
- トークン上限チェック
- Free/Pro プランの区別

**インターフェース**:
- 入力: ユーザー ID、使用トークン数
- 出力: 残りトークン数

---

### 15. プラン管理サービス
**責務**: ユーザーのプラン情報を管理

**主要機能**:
- Free/Pro プランの管理
- プラン切り替え
- AWS Marketplace との連携
- サブスクリプション管理

**インターフェース**:
- 入力: ユーザー ID、プラン情報
- 出力: プラン情報

---

### 16. キャラクター管理サービス
**責務**: キャラクター分岐ロジックを管理

**主要機能**:
- シーン選択に基づくキャラクター分岐
- キャラクター情報の取得
- キャラクターリセット

**インターフェース**:
- 入力: シーン選択結果
- 出力: キャラクター情報

---

### 17. 攻撃検知サービス
**責務**: プロンプトインジェクションとキャラクター攻撃を検知

**主要機能**:
- プロンプトインジェクション検知
- キーワードベースの攻撃検知
- 攻撃時の対応メッセージ生成

**インターフェース**:
- 入力: ユーザーの入力テキスト
- 出力: 攻撃判定結果、対応メッセージ

---

### 18. 言い訳履歴サービス
**責務**: ユーザーの言い訳履歴を管理

**主要機能**:
- 言い訳の保存
- 言い訳の取得
- 言い訳の削除

**インターフェース**:
- 入力: ユーザー ID、言い訳テキスト
- 出力: 言い訳履歴

---

### 19. API ゲートウェイ
**責務**: API エンドポイントを管理

**主要機能**:
- リクエストのルーティング
- 認証の検証
- レスポンスの形式化
- エラーハンドリング

**インターフェース**:
- 入力: HTTP リクエスト
- 出力: HTTP レスポンス

---

### 20. エラーハンドラー
**責務**: 全エラーを集中管理

**主要機能**:
- エラーのキャッチ
- エラーログの記録
- エラーレスポンスの生成
- エラーの分類

**インターフェース**:
- 入力: エラー情報
- 出力: エラーレスポンス

---

## データ層コンポーネント

### 21. ユーザーテーブル
**責務**: ユーザー情報を保存

**主要機能**:
- ユーザー ID
- メールアドレス
- パスワードハッシュ
- プラン情報
- シーン選択情報

**インターフェース**:
- 入力: ユーザー情報
- 出力: ユーザーレコード

---

### 22. トークン使用量テーブル
**責務**: ユーザーのトークン使用量を保存

**主要機能**:
- ユーザー ID
- 日付
- 使用トークン数
- 残りトークン数

**インターフェース**:
- 入力: トークン使用情報
- 出力: トークン使用量レコード

---

### 23. 言い訳履歴テーブル
**責務**: ユーザーの言い訳履歴を保存

**主要機能**:
- ユーザー ID
- 言い訳テキスト
- 生成日時
- シーン情報

**インターフェース**:
- 入力: 言い訳情報
- 出力: 言い訳履歴レコード

---

## コンポーネント関係図

```
フロントエンド層
├── ランディングページ
├── デモ体験
├── オンボーディング
├── 言い訳生成UI
├── 懺悔室UI
├── サボり肯定ポップアップ
├── キャラクター表示
├── 認証UI
├── 設定UI
└── 共通UIコンポーネント

バックエンド層
├── API ゲートウェイ
├── 認証サービス
├── 言い訳生成サービス
├── 懺悔室サービス
├── トークン管理サービス
├── プラン管理サービス
├── キャラクター管理サービス
├── 攻撃検知サービス
├── 言い訳履歴サービス
└── エラーハンドラー

データ層
├── ユーザーテーブル
├── トークン使用量テーブル
└── 言い訳履歴テーブル
```
