# Unit 1 Functional Design Plan - オンボーディング・認証

## ユニット概要

**Unit 1**: オンボーディング・認証 (Onboarding/Authentication)

**責務**: ユーザーのオンボーディングフロー全体と認証機能を管理

**対象ストーリー**: 1.1 - 1.6
- ストーリー 1.1: ランディングページの表示
- ストーリー 1.2: デモ体験（言い訳生成）
- ストーリー 1.3: アカウント登録
- ストーリー 1.4: プラン選択
- ストーリー 1.5: シーン選択
- ストーリー 1.6: キャラクター表示

**含まれるコンポーネント**:
- フロントエンド: ランディングページ、デモ体験、オンボーディング、認証UI、キャラクター表示、共通UIコンポーネント
- バックエンド: 認証サービス、キャラクター管理サービス
- データ: ユーザーテーブル
